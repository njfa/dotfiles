# 包括的コードレビュー実行

指定されたファイルまたはプロジェクト全体に対して、5つの専門エージェントによる包括的なコードレビューを実行します。

## レビュー対象

レビュー対象: **$ARGUMENTS**

対象が指定されていない場合は、プロジェクト全体をレビューします。

## レビュー実行プロセス

以下の5つの専門エージェントを順次実行し、各観点からの詳細レビューを実施します：

### 1. アーキテクチャ・設計品質レビュー

- **実行エージェント**: `architecture-reviewer`
- **評価観点**: SOLID原則、デザインパターン、モジュール設計、拡張性・保守性
- **アウトプット**: 設計品質スコア、改善提案、リファクタリング推奨箇所

### 2. セキュリティレビュー

- **実行エージェント**: `security-reviewer`
- **評価観点**: 認証・認可、入力値検証、機密情報保護、セキュアコーディング
- **アウトプット**: セキュリティリスクレベル、脆弱性検出、対策優先度

### 3. パフォーマンスレビュー

- **実行エージェント**: `performance-reviewer`
- **評価観点**: アルゴリズム効率性、メモリ使用量、データベースクエリ、ボトルネック
- **アウトプット**: パフォーマンススコア、ボトルネック特定、最適化提案

### 4. テスト品質レビュー

- **実行エージェント**: `test-quality-reviewer`
- **評価観点**: テストカバレッジ、テストケース網羅性、テストコード保守性、モック・スタブ
- **アウトプット**: テスト品質スコア、カバレッジ分析、テスト戦略改善

### 5. コード品質・可読性レビュー

- **実行エージェント**: `code-quality-reviewer`
- **評価観点**: コーディング規約、命名規則、コメント品質、コード複雑度
- **アウトプット**: 可読性・保守性スコア、Clean Code原則遵守度、改善優先度

## 実行指示

**重要**: 以下の5つのエージェントを**並列実行**してください。各エージェントは独立した観点でレビューを行うため、同時実行が可能です。

以下の5つのエージェントを同時に実行してください：

1. **アーキテクチャ・設計品質レビューエージェント**
   - レビュー対象を分析し、設計品質の評価を実施
   - 結果をレポート形式でまとめる

2. **セキュリティレビューエージェント**
   - セキュリティ脆弱性の検出と防御的対策の評価を実施
   - 検出された問題の優先度付けを行う

3. **パフォーマンスレビューエージェント**
   - パフォーマンスボトルネックの特定と最適化案の提示を実施
   - 実行効率とリソース使用量の分析を行う

4. **テスト品質レビューエージェント**
   - テストの網羅性と効果性の評価を実施
   - テスト戦略の改善提案を行う

5. **コード品質・可読性レビューエージェント**
   - Clean Codeの原則に基づく品質評価を実施
   - 可読性と保守性の改善提案を行う

**並列実行のメリット**:

- 実行時間が大幅に短縮（約5倍高速化）
- 各エージェントが独立した観点で偏見のない評価を実施
- リソース効率的な処理

## 実行結果の出力

### ファイル出力要件

**必須**: 全エージェントのレビュー完了後、実行結果を以下のmarkdownファイルに出力してください：

- **出力ファイル名**: `code_review_report_YYYYMMDD_HHMMSS.md`
- **出力場所**: `docs/_review/`ディレクトリ
- **ファイル形式**: Markdown
- **エンコーディング**: UTF-8

### ファイル生成手順

1. `docs/_review`ディレクトリが存在しない場合は作成する
2. `date +"%Y%m%d_%H%M%S"`コマンドを実行してタイムスタンプを取得
3. `docs/_review/code_review_report_{タイムスタンプ}.md`のファイルパスでレポートを作成
4. 以下の統合レポート形式でファイルに出力

## 注意事項

### 実行要件

- **並列実行の要件**: 5つのエージェントを必ず同時実行すること
- 各エージェントは完全に独立して実行し、異なる観点からの評価を確保する
- エージェント間でのコンテキスト共有は行わず、各専門性を維持する

### 出力要件

- **ファイル出力必須**: 統合レポートを必ずmarkdownファイルに出力すること
- **ファイル名形式**: `code_review_report_YYYYMMDD_HHMMSS.md`
- **出力場所**: `docs/_review/`ディレクトリ
- **ディレクトリ作成**: `docs/_review`ディレクトリが存在しない場合は自動作成
- **各エージェントの詳細結果**: 統合レポート内に全エージェントの詳細出力を含めること

### 品質要件

- 並列実行完了後に統合レポートで全体的な優先度と実行可能性を考慮した提案を行う
- レビュー結果は建設的で実装可能な改善策に焦点を当てる
- 並列実行により大幅な時間短縮と効率性の向上を実現する
- 生成されたレポートファイルはチーム内で共有・議論可能な形式にする
