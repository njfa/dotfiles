# test-code-implement

テスト駆動開発を効率的に進めるための統合コマンドです。指定された指示内容に基づいてテストコード実装とプロダクションコード修正を自動化します。

## 使用方法

```
/test-code-implement [指示内容]
```

## 処理フロー

### 0. 事前チェック

- **依存ライブラリの確認**: テストフレームワークと必要なライブラリの存在確認
- **不足時の即座停止**: 依存関係が不足している場合は処理を即座に停止し、ユーザーに詳細をフィードバック

### 1. 作業計画の策定

- 指定された指示内容を分析
- テストコード作成対象ファイルの特定
- 作業内容と優先順位の決定

### 2. パイプライン並列処理

- 各対象ファイルに対してtest-code-implementerエージェントを並列実行
- **完了次第即座開始**: テストコード実装が完了したファイルから順次以下を実行
  - 個別ファイルのテスト実行
  - テスト結果の分析
  - 修正必要時はproduction-code-implementerエージェントに即座依頼
- 全てのテスト-プロダクションペアを独立して並列処理

### 3. 全体テスト実行

- 全てのパイプライン処理完了を待機
- 全テストコードを対象とした統合テスト実行
- テスト結果の評価と問題点の特定

### 4. 後処理と結果報告

#### 4-1. プロダクションコード修正対応

- **修正必要時**: production-code-implementerエージェントに修正を依頼
- **テスト実行**: 修正後のテストコード実行

#### 4-2. テストコード再修正対応  

- **再修正必要時**: test-code-implementerエージェントに修正を依頼
- **テスト再実行**: 修正後のテストコード再実行

#### 4-3. 最終結果報告

- **成功・失敗問わず**: テスト実行結果をユーザーに詳細報告
- **実装サマリー**: 作成・修正されたファイルの概要
- **残課題**: 未解決の問題がある場合の詳細情報

## 入力パラメータ

- **指示内容** (必須): テスト実装の指示や要件
  - 例: "ユーザー認証機能のテストを実装"
  - 例: "API エンドポイントのテストカバレッジを向上"

## 実行例

```bash
/test-code-implement "ユーザー管理機能のテストを作成し、既存のプロダクションコードの問題を修正"
```

## 特徴

- **事前依存関係チェック**: 処理開始前に必要なライブラリの存在を確認
- **パイプライン並列処理**: テスト完了次第即座にプロダクション修正を開始
- **最大並列度**: 待機時間を最小化した効率的な処理フロー
- **サブエージェント活用**: 全ての修正作業を専門エージェントに委譲
- **確実な結果報告**: 成功・失敗問わず詳細な実行結果を提供
- **統合テスト**: 個別テストと全体テストの両方を実行

## エラーハンドリング

### 依存関係不足時のフィードバック例

```
❌ 依存関係エラー: テスト実装に必要なライブラリが不足しています

不足しているライブラリ:
- Jest (テストフレームワーク)
- @testing-library/react (React テストユーティリティ)

インストール方法:
npm install --save-dev jest @testing-library/react

または

yarn add --dev jest @testing-library/react

インストール完了後、再度コマンドを実行してください。
```

## 注意事項

- 処理開始前に依存関係の確認が行われます
- プロダクションコードとテストコードの依存関係を事前に確認してください
- 全ての修正作業は専門エージェントが実行し、結果は必ずユーザーに報告されます
