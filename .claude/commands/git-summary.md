# Git変更内容サマリ生成

リポジトリの変更内容を分析し、以下の情報を含む構造化されたサマリレポートを生成します：

## 分析対象

分析期間: **$ARGUMENTS**

引数が指定されていない場合は、最新のコミットから作業ディレクトリまでの変更を分析します。

## サマリ生成プロセス

### 1. Git状態確認

- `git status`で現在の作業ディレクトリ状態を確認
- `git branch`で現在のブランチ情報を取得

### 2. 変更差分取得

- `git diff HEAD`でコミットされていない変更を取得
- `git diff --staged`でステージングされた変更を取得
- `git log --oneline -10`で最近のコミット履歴を取得

### 3. 変更統計

- `git diff --stat HEAD`で変更ファイルの統計情報を取得
- `git diff --numstat HEAD`で詳細な行数統計を取得

### 4. ファイル別変更詳細

変更されたファイルごとに以下を分析：

- 変更の種類（追加/削除/修正）
- 変更行数
- 変更内容の概要

## 実行手順

1. **Git状態の確認**

   ```bash
   git status
   git branch
   git log --oneline -10
   ```

2. **変更差分の取得**

   ```bash
   git diff HEAD --stat
   git diff HEAD --numstat
   git diff HEAD
   git diff --staged
   ```

3. **レポート生成**
   - 取得した情報を上記フォーマットに整理
   - ファイル別の変更内容を分析・分類
   - 統計情報の計算と集計

4. **出力**
   - コンソールに整理されたサマリを表示
   - オプション: `git_summary_YYYYMMDD_HHMMSS.md`ファイルとして保存

## 使用例

```bash
# 現在の変更をサマリ
/git-summary

# 特定のコミットからの変更をサマリ
/git-summary HEAD~5

# 特定のブランチとの差分をサマリ
/git-summary main..feature-branch
```

## 注意事項

- 大量の変更がある場合は、ファイル数制限を設けて要約する
- バイナリファイルの変更は適切にフィルタリングする
- 機密情報を含む可能性のある変更は注意深く扱う
